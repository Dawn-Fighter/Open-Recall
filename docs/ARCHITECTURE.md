# Architecture

The agent is intentionally small: Streamlit provides the operator interface,
`incident_agent` owns the response workflow, Hindsight stores incident memory,
and CascadeFlow/Groq routing is represented as an auditable runtime trace.

## Flow

```text
Raw alert
  -> normalize incident object
  -> classify SRE / Security / Hybrid
  -> recall similar memories from Hindsight or local seed data
  -> route RCA generation through cheap or strong model path
  -> produce verification-first runbook
  -> retain final RCA and lessons after resolution
```

## Components

- `app.py` renders the Streamlit cockpit, metrics, memory recall, runbook,
  audit trace, and learning loop.
- `incident_agent/models.py` defines the typed incident, memory, trace, and
  analysis result objects.
- `incident_agent/memory.py` adapts to open-source Hindsight and falls back to
  deterministic local JSON memory when Hindsight is unavailable.
- `incident_agent/router.py` records CascadeFlow-style routing decisions and can
  call Groq when `CASCADEFLOW_LIVE_GROQ=true`.
- `incident_agent/workflow.py` coordinates normalization, classification,
  memory recall, RCA merge logic, evidence preservation, and postmortem output.
- `data/seed_incidents.json` and `data/sample_alerts.json` provide repeatable
  demo data.

## Runtime Modes

Fallback mode is the default safe mode. It uses seed incidents and deterministic
RCA generation, so reviewers can run the project without Docker, Hindsight, or
Groq credentials.

Live demo mode connects to Hindsight and allows Groq model calls. Use this for
recordings or judging when you need to prove the external integrations.

## Data Boundaries

`data/seed_incidents.json` is safe synthetic demo data. `data/local_memory.json`
is generated by the learning loop and is ignored because it can contain local
incident details. `.env` and `.streamlit/secrets.toml` are ignored for the same
reason.
